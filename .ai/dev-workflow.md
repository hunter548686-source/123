# 开发执行工作流

## 一、本文档只描述“如何开发这个网站”

本文件描述的是开发过程中的协作方式，不是网站正式运行方式。

## 二、固定开发角色

- 规划：`GPT-5.4`
- 开发执行：本地 `Ollama + qwen2.5-coder`
- 审查：`GPT-5.4`
- 本地环境：`WSL + Windows Ollama API`

## 三、固定开发闭环

1. `GPT-5.4` 先规划
2. 本地 `qwen2.5-coder` 执行代码修改
3. `GPT-5.4` 审查
4. 审查不通过则继续给本地模型修复指令
5. 审查通过后继续测试、整理文档、必要时部署

## 四、适用范围

这套流程只用于：

- 开发网站代码
- 本地修复 bug
- 本地生成 patch / diff
- 本地开发审查链

不用于：

- 用户任务正式执行
- 网站对外算力运行
- 网站生产环境交付
