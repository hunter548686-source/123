# UI 变更

## 首页（成品态）
- 首页内容从“计划/流程说明”升级为“产品交付首页”，以用户结果为中心展示价值。
- 新增交付指标卡（交付时长、成功率、可用 provider、成本可视化覆盖）。
- 指标卡从静态值升级为实时 API 数据：`GET /api/home/metrics`。
- 新增交付闭环步骤区（创建任务、自动调度、失败迁移、结果归档）。
- 新增系统入口群组（控制台、任务列表、账单中心、运维后台）。

## 任务详情页
- 产物区增加“下载”按钮（调用后端下载解析接口）。
- 展示 `downloadUrl` 与 `checksum`（可用时）。
- 下载动作反馈提示。

## 管理后台页
- 增加监控总览展示：
  - adapter key / marketplace name
  - active runs / retry queue
  - pending cleanup / open cancellations
  - provider cost / runtime seconds

## 后台任务台（升级）
- 任务台切换为后台真实接口（全量任务，不再是当前登录用户任务）。
- 支持状态筛选、手动重试、手动取消、跳转后台任务详情。
- 显示后台汇总统计（总任务 / 运行中 / 失败 / 已完成）。

## 后台任务详情（新增）
- 新增页面：`/admin/tasks/[taskId]`
- 展示状态、审查结论、Provider/GPU、报价/实扣/成本。
- 展示 run 历史、事件日志、产物下载入口。
- 支持后台直接触发重试与取消。

## 后台用户与权限（新增）
- 新增页面：`/admin/users`
- 展示用户角色、状态、钱包余额、任务统计与注册时间。
