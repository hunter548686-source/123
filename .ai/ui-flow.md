# UI 流程说明

## 一、前台用户流

### 首页

目标：

- 让用户理解平台卖的是“稳定交付”
- 强调自动比价、自动选机、自动重试、失败迁移

### 控制台

展示：

- 任务概览
- 当前运行状态
- 成本概览
- 最近交付结果

### 新建任务页

流程：

1. 选择任务类型和模板
2. 输入 prompt 与参数
3. 获取报价
4. 创建任务

### 任务详情页

展示重点应是正式产品运行信息：

- 当前阶段
- 当前 provider
- 历史运行尝试
- 失败恢复
- 成本
- 结果归档

## 二、后台管理流

### `/admin`

这是运营 / 管理后台。

应分成两类区域：

#### 1. 产品运行区

展示：

- 任务总量
- 运行中任务
- 失败任务
- provider 健康
- 成本与利润

#### 2. 开发 / 调试工具区

展示：

- 本地代码执行器
- diff preview
- rollback
- 开发审计记录

这一块必须明确标注为开发 / 调试工具，不代表用户任务正式运行方式。

### `/admin/tasks`

展示：

- 正式任务状态
- provider 选择结果
- 审查摘要
- 异常与失败原因
- 手动重试 / 取消操作
- 任务详情跳转入口

### `/admin/providers`

展示：

- 外部 provider 报价
- 健康度
- 成功率
- 最近抓取时间

### `/admin/tasks/[taskId]`

展示：

- 单任务全链路细节（状态、审查、成本）
- run 历史、事件日志、产物归档
- 后台手动干预操作（重试 / 取消）

### `/admin/users`

展示：

- 用户角色与状态
- 钱包余额与冻结金额
- 任务总量与完成/失败分布
